# thermal-phosphor

The **constructive-mechanics lineage**: an anisotropic mover + one constraint (a compass) + **mass** —
built up, step by step, toward a self-templating replicator. It asks where a substrate *comes from* —
one floor below where the [character-framework](https://github.com/ronviers/character-framework)
begins (character takes the rate-graph as given; thermal-phosphor builds it).

## Scope — a construction, not a history

thermal-phosphor asks **how to build** a self-templating replicator from two tools up. It does **not**
ask **how life historically arose** on Earth. Those are different questions with likely different
answers, and the actual history may be unknowable — so the model carries no commitment about *when*,
*how often*, or *why once*. Uniqueness (n=1), the last universal common ancestor, hard-step timing,
and geological before/after boundaries are questions about the *record*, not about *buildability*, and
they are out of scope by design. The deliverable is a **blueprint that closes** — not a reconstruction
of what happened.

The central design target is **strong closure**: a loop whose copying *fidelity* rides on the copied
object itself, so improvement is stored where it can accumulate — as opposed to a channel whose
fidelity lives in the surrounding medium and can only be sorted, never climbed.
